# 两级意图分类 — 改进方案

> 基于「小白编码 Agent」定位，将现有 7 类扁平意图升级为 6 大类 × 17 子类（含游戏）的两级分类体系。

---

## 一、两级分类矩阵

```
Level1 (大类, 6 选 1)          Level2 (子类, 16 选 1)              命中后行为
──────────────────────────────────────────────────────────────────────────
learn (学习理解)    ─┬─ explain      解释概念/知识点          文字回复
                     ├─ debug        排查报错(带代码)         文字 + 错误标注
                     ├─ compare      技术对比/选型建议         文字 + 对比表格
                     └─ casual       日常闲聊/问好             文字回复

code  (编码实战)    ─┬─ snippet      写函数/算法片段           代码块
                     ├─ component    写 UI 组件                代码块 + 组件模板
                     ├─ fix          修复 Bug                  diff 模式代码块
                     └─ refactor     重构优化                  代码块 + 建议列表

build (建站/游戏)   ─┬─ page         生成单页/落地页           预览 + 产物
                     ├─ site         生成完整多页站            预览 + 产物
                     ├─ modify       修改已有网站(含上下文)    预览 + 产物
                     └─ game         生成互动小游戏(无框架→Three.js) 预览+产物+操作

doc   (文档方案)    ─┬─ readme       生成 README              Markdown 流式卡片
                     ├─ tutorial     生成教程/指南             Markdown 流式卡片
                     └─ plan         生成方案/计划/设计文档     Markdown 流式卡片

translate (翻译转换) ─┬─ text         翻译文本                  文字回复
                     └─ code_lang    跨语言代码翻译            代码块

unsupported(不支持)  ─ 无子类         提示"暂不支持" + 统计     弹出提示
```

---

## 二、分类 prompt（一次 LLM 调用出两级）

```
你是小白编码助手的意图分类器。根据用户输入，返回 JSON：
{"level1": "...", "level2": "...", "confidence": 0.0~1.0}

Level1(6选1):
- learn: 学习、提问、理解概念、排查错误、对比技术、闲聊
- code: 编码要求(手写代码、生成函数、修改逻辑、重构优化)
- build: 生成网站页面、修改已有网站、生成互动游戏
- doc: 生成文档、方案、教程
- translate: 翻译或格式转换
- unsupported: 以上都不匹配

Level2(只选对应 Level1 下的，共 16 种子类):

learn 下:
  explain → 解释概念/知识点(无代码报错)
  debug   → 排查报错/异常(含错误信息或代码)
  compare → 技术对比/选型建议(A vs B)
  casual  → 日常闲聊/打招呼/无技术内容

code 下:
  snippet   → 写函数/算法片段(<50行)
  component → 写 UI 组件(按钮/表单/卡片等)
  fix       → 修复 Bug(用户贴了有问题的代码)
  refactor  → 重构优化(改进已有代码质量)

build 下:
  page   → 生成单页/落地页("主页"/"landing")
  site   → 生成完整网站("官网"/"商城"/"后台")
  modify → 修改已有网站(会话中有之前生成的 HTML)
  game   → 生成互动小游戏("游戏"/"贪吃蛇"/"打砖块"/"坦克大战")，不指定框架默认用 Three.js

doc 下:
  readme   → 生成 README / 项目文档
  tutorial → 生成教程/指南/步骤说明
  plan     → 生成方案/设计文档/架构计划

translate 下:
  text      → 自然语言翻译(中→英等)
  code_lang → 代码跨语言翻译(Python→JS等)

用户输入: {user_message}
```

Token 估算：~250 输入 + ~25 输出，耗时 < 1.2s。

## 三、现有 Skill 适配对照

| 现有 Skill | 当前触发意图 | 改进后触发（level1/level2） |
|-----------|-------------|---------------------------|
| `explain` | chat | learn/explain, learn/debug, learn/compare, learn/casual, translate/text |
| `write_code` | code | code/snippet, code/component, code/fix, code/refactor, translate/code_lang |
| `generate_site` | generate, modify | build/page, build/site, build/modify, build/game |
| `generate_doc` | doc | doc/readme, doc/tutorial, doc/plan |
| (无) | unsupported | unsupported |

**结论**：现有 4 个 Skill 完全覆盖 15 种有效子意图，不需要新增 Skill。但需要在 Skill handler 中根据 `level2` 调整 System Prompt 模式（见下文 §五）。

## 四、需要新增/修改的内容

### 4.1 后端

| 文件 | 改动 |
|------|------|
| `intent_classifier.py` | prompt 改为两级输出；classify 返回 `{level1, level2, confidence}`；上下文捷径保留（session 有 HTML → level1=build level2=modify） |
| `router.py` | `INTENT_SKILL_MAP` 改为 `(level1, level2) → skill` 二维映射；`INTENT_LABELS` 加两级标签 `{learn/explain: "学习理解·概念解释", ...}` |
| `runner.py` | intent 事件透传 `level1`/`level2` 字段（供前端分级显示） |
| `events.py` | intent 事件数据结构加 `level1`/`level2` 字段 |
| `analytics.py` | 意图统计从 `an:intent:hit:{intent}` 改为 `an:intent:hit:{level1}:{level2}` 两级分层计数 |
| `explain.py` | 根据 `level2` 切换 prompt 模式：debug → 错误诊断模式；compare → 对比分析模式 |
| `write_code.py` | 根据 `level2` 切换 prompt 模式：fix → diff 模式；refactor → 质量分析模式 |

### 4.2 前端

| 文件 | 改动 |
|------|------|
| `types.ts` | IntentEvent 加 `level1: string` / `level2: string` |
| `chat.ts` | ChatCallbacks onIntent 透传 level1/level2 |
| `ThoughtTrail.vue` | 意图标签改为两级显示，如 "🧠 编码实战 → 修复 Bug" |
| `ChatView.vue` | currentIntent 存储 `{level1, level2}` 代替单一字符串 |
| `AdminView.vue` | 系统分析-意图命中率表改为两级展开（Level1 行可展开显示子意图） |

## 五、Skill handler 按 level2 调模式

### 5.1 explain.py — 根据 level2 切换 System Prompt

```python
PROMPTS = {
    "explain":  "你是编程入门导师，用通俗易懂的语言解释概念，多举例。",
    "debug":    "你是资深 debug 专家。根据用户贴的错误信息给出根因、修复方案和预防建议。",
    "compare":  "你是技术选型顾问。对比用户提到的技术方案，列出优劣和适用场景。",
    "casual":   "你是友好的编程助手，轻松回答用户的问题。",
    "text":     "你是翻译专家。把用户提供的文本准确翻译。",
}
```

### 5.2 write_code.py — 根据 level2 输出不同格式

```python
PROMPTS = {
    "snippet":    "你是算法专家。只输出内联代码，带极简注释。",
    "component":  "你是前端组件专家。输出完整组件代码，含样式和使用示例。",
    "fix":        "你是 Bug 修复专家。用 diff 格式标注修复，说明问题根因。",
    "refactor":   "你是代码质量专家。输出重构后的代码，附带改进点列表。",
    "code_lang":  "你是跨语言翻译专家。把代码逐行翻译到目标语言，保持逻辑不变。",
}
```

### 5.3 generate_site.py — build/page 与 build/site 的区别

| 子意图 | 差异 | Coder Prompt 侧重 |
|--------|------|------------------|
| `page` | 单页，不含导航/多路由 | 单文件 HTML，内联 CSS/JS，响应式 |
| `site` | 多页，需导航/子页链接 | 主文件 + 结构说明，多页面布局 |
| `modify` | 已有旧 HTML，只改部分 | 保持原结构，只修改用户要求的区域 |
| `game`   | 互动游戏，canvas/WebGL | Three.js CDN + 完整游戏循环 + 计分/关卡 + 键盘/触屏操作 |

### 5.4 游戏生成策略（新增）

当意图为 `build/game` 时：

**框架选择逻辑**：
- 用户指定了引擎/框架（"用 Phaser 写"）→ 按指定框架
- 用户未指定 → **默认 Three.js**（能力最强、生态最好、CDN 直链、iframable）

**Coder Prompt 侧重**：
```
你是一个游戏开发者。生成一个完整的单文件 HTML 互动游戏。
必须包含: <script src="https://cdn.jsdelivr.net/npm/three@0.160.0/build/three.min.js"></script>
游戏要素: 3D 场景 + 玩家控制 + 碰撞检测 + 计分系统 + 开始/重新开始按钮 + 操作提示
兼容: 键盘(PC) + 触屏(手机)操作
```

**与普通 site 的区别**：
| | page/site | game |
|--|-----------|------|
| 主要技术 | HTML+CSS+JS | Three.js Canvas |
| 产出 | 静态页面 | 可交互游戏 |
| 预览表现 | 滚动浏览 | 点击/键盘操作 |
| 产物文件 | index.html | index.html + 可能 assets/ |
| 修改方式 | 改文案/布局 | 改关卡/参数/逻辑 |

**关键词兜底表**：
- 一级判 `build`：含 "游戏" / "game" / "贪吃蛇" / "打砖块" / "坦克大战" / "射击" / "2048" / "消消乐"
- 二级细判 `game`：上述关键词 + "玩" / "控制" / "关卡" / "得分"
- 注意区分：含 "做游戏" 的教程类 → `doc/tutorial`，不是 `build/game`

## 六、存量兼容 & 平滑迁移

| 措施 | 说明 |
|------|------|
| 旧代码兼容 | `onIntent` 的 label 用 `"level1_label · level2_label"` 拼接，前端向下兼容 |
| 统计字段兼容 | 旧 `an:intent:hit` 保留当日日志，新键 `an:intent:hit:{l1}:{l2}` 从此分段 |
| 关键词兜底保留 | LLM 分类器失败时，关键词表从 Level1 + Level2 双级匹配 |
| 上下文捷径不变 | 会话已有 HTML → 强制 `build/modify`，跳过 LLM 分类 |
