## 一、现状总览（v0.8.6 代码实测）

| 层面 | 已有 | 缺口 |
|------|------|------|
| 意图判定 | 5 模块并行 + 安全优先汇总 | 语义只吃 `last[:500]`；上下文只取最近 1 条；用户偏好缺席 |
| 闲聊方向 | `explain`/`search_agent` + QC 三裁判评分 | QC 只评分不纠错；无低分重答 |

## 二、AI 核心流程重构（14 项决策已确认）

### 2.1 新流程四步（2026-07-23 终版）

1. **上下文融合**：用户输入 + 最近 5 条（Redis L1→MySQL L0 回退）+ 项目关键词 + **Chroma project_memory** + **Chroma user_preferences** → 语义判断
2. **执行计划**：多任务拆分 → 闲聊/建站分流；不确定仅高风险反问
3. **闲聊方向**：安全校验 → LLM → 多模型打分 → 低分轻量重答 1 轮
4. **建站方向**：安全校验 → **Planner→Coder→Reviewer→Reflexion（加阶段标签+自评）** → 打分 → 低分定位失败阶段 → 只重启该阶段（保留上游，≤3 轮 / 30s，兼容中断续跑）→ best-of-N

## 三、产品记忆压缩（四层架构）

### 3.1 四层总览

| 层 | 内容 | 存储 | 生命周期 | 写入时机 |
|----|------|------|----------|----------|
| L0 | 完整对话 | MySQL `Message` | 永久 | 每条消息落库 |
| L1 | 滚动摘要 ≤200 字 | Redis `summary:{conv_id}` **1d TTL** | 1 天，过期从 MySQL 重压 | 每 6 条 + done 后刷新 |
| L2 | **精炼后对话** | MySQL `Message.content`（覆写 assistant） | 永久 | done 后，LLM 精炼原始流式文本 |
| L2+ | 结构化项目记忆 | MySQL `project_memories` + Chroma `project_memory` | 永久（compaction 防膨胀） | done 后，LLM 从 L2+产物抽取 |
| L3 | 用户偏好卡片 | `User.card_json` + Chroma `user_preferences` | 永久 | done 后蒸馏 + 用户手动设置 |

### 3.2 L1 过期回退流程

```
get_summary(conv_id)
  → Redis GET summary:{conv_id}
  → HIT → 返回摘要
  → MISS (过期) →
      SELECT content FROM Message WHERE conversation_id=? ORDER BY id DESC LIMIT 30
      → LLM 压缩为 ≤200 字摘要
      → SETEX summary:{conv_id} 86400 <新摘要>
      → 返回摘要
```

### 3.3 L2 对话精炼流程（在 done 钩子中执行）

```
1. 取本轮 raw_assistant_text（SSE 流式拼接的完整文本）
2. refine_prompt = "去掉重复/口头语/冗余，保留完整信息，结构清晰，≤300字"
3. LLM(deepseek) ainvoke → refined_text
4. UPDATE Message SET content=refined_text, refined=1 WHERE msg_id=? （幂等：refined=1 则跳过）
5. maybe_compress_summary(conv_id, latest_user, refined_text) → 更新 L1
6. 可选 SSE 事件 {"type":"refined","msg_id":?,"content":"..."} 供前端切换展示
```

## 四、Chroma 六集合详细规范（核心新增）

> 六集合按隔离维度和使用频率设计。每个集合独立建库，metadata 使用 Chroma 原生 `where` 过滤，不设 collection 级前缀（保持查询简洁）。

### 4.1 集合一览

| 集合 | 用途 | 隔离字段 | 写入频率 | 检索场景 |
|------|------|----------|----------|----------|
| `conversation_context` | 对话上下文相似度 | `conversation_id` | 每条消息（高） | `find_relevant_messages` |
| `user_preferences` | 用户偏好/禁忌/习惯 | `user_id` | 每轮 done（低） | step1 送 intent |
| `project_memory` | 项目决策/约束/需求历史 | `project_id` + `user_id` | 每轮 done（中） | step1/Planner/Coder |
| `project_code` | 大项目代码语义分块 | `project_id` | 每次代码变更（高） | Coder/Reflexion |
| `error_patterns` | 全局错误模式 | 无（全局） | 修复确认后（低） | repair 阶段 |
| `components` | 公共组件库 | 无 | 不变（已有） | `build_rag_context` |

### 4.2 各集合精确 Schema

#### 4.2.1 `conversation_context`（已有，不改）

```
id:           "msg_{msg_id}"
document:     content[:2000]
metadata:     { conversation_id:int, role:"user"|"assistant", msg_id:int }
检索:         where={"conversation_id": conv_id}, n_results≤10
阈值:         similarity ≥ 0.55 (余弦)
```

#### 4.2.2 `user_preferences`（新建）

```
id:           "upref_{user_id}_{type}_{hash4}"    # 例: upref_42_style_a3f2
document:     content (偏好描述文本，用于语义检索)
metadata:     { user_id:int, type:"style"|"constraint"|"habit", importance:1-5,
               source:"distill"|"manual"|"inferred", ts:unix }

写入(UPSERT):
  每次 done → distill_user_preferences(user_id, refined_text) → 返回 [{type,content,importance}]
  → 对每条 upsert:
      col.upsert(ids=["upref_{uid}_{t}_{hash}"], documents=[content], metadatas=[{...}])

检索:
  where={"user_id": user_id}, query_texts=[current_input], n_results=5
  → 返回 top-k 偏好，拼进 intent step1 prompt

Compaction:
  user_preferences 累计 > 30 条 → LLM 合并同 type → 精简到 ≤10 条
  (低频，用户级别，非每轮)
```

#### 4.2.3 `project_memory`（新建）

```
id:           "pmem_{project_id}_{type}_{hash4}"   # 例: pmem_7_decision_b1c0
document:     content (决策/约束/需求描述，用于语义检索)
metadata:     { project_id:int, user_id:int,
               type:"decision"|"constraint"|"requirement"|"artifact"|"fact",
               importance:1-5, ts:unix }

写入(UPSERT):
  每次 done → distill_project_memory(project_id, user_id, refined_text, artifacts)
  → 返回 [{type,content,importance}] → upsert

检索:
  where={"project_id": project_id}, query_texts=[current_input], n_results=5
  → 注入 Planner prompt + Coder prompt

Compaction:
  单项目 memories > 50 条 → LLM 合并 → 精简覆盖
  或按 age > 30 天触发（cron / 后台任务）
```

#### 4.2.4 `project_code`（新建）

```
id:           "pcode_{project_id}_{file_path}_{chunk_hash}"   # 复合 id 幂等
document:     code_chunk (函数/区块级代码片段)
metadata:     { project_id:int, file_path:str, function_name:str|None,
               language:"html"|"css"|"js"|..., chunk_type:"function"|"style_block"|"script",
               chunk_hash:str, ts:unix }

写入(UPSERT，每次代码变更):
  1. parse_code_tree(html) → [{file_path, function_name, chunk_text, chunk_hash}]
  2. 对每个 chunk: 计算 hash → 查 Chroma id 是否已存在
     - 已存在且 hash 相同 → skip
     - 已存在但 hash 不同 → upsert (覆盖)
     - 不存在 → insert
  3. 删除当前 project_id 下 hash 已不存在的旧条目（可选，或设 TTL）

检索:
  where={"project_id": project_id}, query_texts=[task_description], n_results=8
  → 注入 Coder prompt: "以下是与当前需求最相关的已有代码块: ..."

触发时机:
  - 每次 generate_site done → 后台异步全量索引 (首次)
  - 每次修改代码 → 增量 upsert (后续)
  - 非阻塞，失败仅 warn
```

#### 4.2.5 `error_patterns`（新建，全局）

```
id:           "err_{error_type_hash}"   # 例: err_a3f2b1c0
document:     trigger_pattern + " → " + fix_pattern
metadata:     { error_type:str, trigger_pattern:str, fix_pattern:str,
               language:"html"|"css"|"js"|"general",
               success_count:int, last_seen_ts:unix }

写入(UPSERT，修复确认后):
  条件: repair 成功 + QC overall ≥ 前一版
  → 查 Chroma 是否有同 error_type (strict match)
    - 有 → upsert: success_count += 1, last_seen_ts = now
    - 无 → insert: success_count = 1

检索:
  query_texts=[error_description], n_results=5
  → 注入 repair prompt: "以前类似错误的修复方式: ..."

种子数据(冷启动)，约 20-30 条:
  {"error_type":"flex overflow","trigger":"display:flex without min-width:0",
   "fix":"add min-width:0 or overflow:hidden to flex child","language":"css"}
  {"error_type":"z-index stacking","trigger":"z-index without positioning context",
   "fix":"ensure parent has position:relative and non-auto z-index","language":"css"}
  ... (见 §7.3)
```

#### 4.2.6 `components`（已有，不变）

```
保持现有: seed_components() → 公共组件库，检索注入 Planner
```

### 4.3 Chroma API 调用模式（规范 Python 伪代码）

```python
import chromadb
from chromadb.utils import embedding_functions

client = chromadb.HttpClient(base_url=settings.chroma_url)
ef = _ef()  # Qwen > DeepSeek > sentence-transformers

# --- 通用检索 ---
def retrieve(collection: str, query: str, where: dict, top_k: int = 5) -> list[dict]:
    col = client.get_or_create_collection(name=collection, embedding_function=ef)
    res = col.query(query_texts=[query], n_results=top_k, where=where)
    return [
        {"id": id_, "content": doc, "metadata": meta, "score": 1 - dist}
        for id_, doc, meta, dist in zip(
            res["ids"][0], res["documents"][0],
            res["metadatas"][0], res["distances"][0]
        )
    ]

# --- 通用 upsert ---
def upsert(collection: str, ids: list, docs: list, metas: list[dict]):
    col = client.get_or_create_collection(name=collection, embedding_function=ef)
    col.upsert(ids=ids, documents=docs, metadatas=metas)
```

**关键原则**：所有用户数据集合必须在 `where` 中带隔离字段（`user_id` / `project_id`），公共集合除外。
