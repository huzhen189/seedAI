# v0.8.0 意图管道重构方案

> 三合一方案：意图多维度并行判断 + 目录结构重构 + Agent/Skill 注册合并

---

## 一、背景与目标

### 现状问题
1. **意图分类单一**：`classify()` 一次 LLM 调用出所有结果，失败降级关键词，无多维度交叉验证
2. **目录扁平混乱**：28 个文件平铺在 `app/` 下，意图/路由/技能/工具混在一起
3. **Agent/Skill 双维护**：`agents.py`（手工列表）和 `skills/*.py`（`register_skill`）维护两份重叠元数据

### 目标
```
意图:  单次LLM → 5模块并行 + 汇总器决策
目录:  28平铺 → 7个功能包
Agent: 双维护 → register_skill 单一来源
```

## 三、意图管道重构

### 3.1 架构对比

```
现状:
  messages → classify() → {level1, level2, confidence, industry}
               ↓ 失败
           _keyword_fallback()

目标:
  messages ─┬→ [规则模块]     0ms   关键词/格式校验
            ├→ [语义模块]     2s    LLM意图分类
            ├→ [上下文模块]   0ms   历史关联修正
            ├→ [工具模块]     0ms   skill能力映射
            └→ [安全模块]     0ms   权限/风险/确认检查
                   ↓
            [汇总器]  优先级决策 + 证据融合 + 置信度排序
                   ↓
            PipelineResult {intent, plan, risk, tools[], evidence}
```

### 3.2 模块 1：规则模块 (`intent/rules.py`)

```python
# 职责: 零延迟关键词/格式检测, 不调LLM
# 输入: 用户消息文本
# 输出: RuleResult

class RuleResult:
    keywords:      list[str]     # 命中的关键词 ["建站", "网站"]
    pattern:       str           # 规则分类: build|learn|code|doc|unsupported
    confidence:    float         # 0.7 (规则固定置信度)
    industry:      str           # 从关键词检测的行业 "restaurant"

# 实现: 复用现有 _keyword_fallback() + 扩展
# 关键词匹配规则表(已有100+词, 保持在 intent_classifier.py 中的定义)
```

### 3.3 模块 2：语义模块 (`intent/semantic.py`)

```python
# 职责: LLM 深层语义理解, 核心意图判断
# 输入: messages + model_id + context_hint
# 输出: SemanticResult

class SemanticResult:
    level1:        str           # build
    level2:        str           # site
    industry:      str           # restaurant
    confidence:    float         # 0.95
    checkpoint_relation: str     # none|resume|correct|override
    raw_output:    str           # LLM 原始返回(审计用)
    latency_ms:    float         # 调用耗时

# 实现: 复用现有 classify_async(), 抽离到独立文件
# 异步调用 chat.ainvoke() → 解析 JSON → 返回
```

### 3.4 模块 3：上下文模块 (`intent/context.py`)

```python
# 职责: 结合历史对话修正意图(确定用户是在追问/纠正/换话题)
# 输入: messages + conversation_id + frontend_hint
# 输出: ContextResult

class ContextResult:
    has_context:   bool          # 是否有历史上下文
    hint:          str           # "天气查询中的城市补充"
    correction:    dict | None   # 对意图的修正(可选)
                                 # {level1: "build", reason: "上条是建站需求"}
    source:        str           # "webllm"|"chroma"|"fallback"|"none"

# 实现: 复用现有 detect_context(), 抽离到独立文件
# 3层兜底: WebLLM → Chroma → 关键词
```

### 3.5 模块 4：工具模块 (`intent/tools.py`)

```python
# 职责: 根据意图+置信度匹配可用技能, 按置信度排序
# 输入: intent + industry + project_status
# 输出: ToolResult

class ToolResult:
    skills:        list[SkillCandidate]  # 按置信度降序
    fallback:      str                   # "explain" 兜底

class SkillCandidate:
    name:          str         # "requirement_agent"
    confidence:    float       # 0.85
    reason:        str         # "意图匹配: build/site"
    requires_doc:  bool        # 是否需要需求文档

# 实现: 查 INTENT_SKILL_MAP + 状态路由(draft→requirement_agent)
# 多候选生成规则:
#   置信度 > 0.8 → 直接路由
#   置信度 0.5-0.8 → 路由 + 带警告
#   置信度 < 0.5 → 出多选项(options)
```

### 3.6 模块 5：安全模块 (`intent/safety.py`)

```python
# 职责: 权限检查/风险检测/二次确认判断
# 输入: intent + messages + user_role
# 输出: SafetyResult

class SafetyResult:
    risk_level:       str     # "low"|"medium"|"high"|"critical"
    requires_confirm: bool    # 是否需要用户二次确认
    permissions_ok:   bool    # 权限是否通过
    block_reason:     str     # 拦截原因(如有)
    risk_tags:        list    # ["delete", "敏感数据", "支付"]

# 风险判断规则:
#   critical: 删除操作/支付/敏感数据 → 直接拦截
#   high:    修改已发布站点/管理操作 → 二次确认
#   medium:  新功能开发 → 正常流程
#   low:     查询/说明/闲聊 → 无需确认
#
# 关键词风险表:
#   删除|清空|drop|rm → critical
#   支付|付款|充值|订单 → critical
#   发布|上线|deploy → high
#   修改|改|modify → medium
```

### 3.7 汇总器 (`intent/pipeline.py`)

```python
# 职责: 并行调用 5 模块 → 优先级决策 → 融合证据 → 输出最终决策
# 入口: classify_v2(messages, model_id, ...) → PipelineResult

class PipelineResult:
    intent:          dict      # {level1, level2, confidence, industry}
    plan:            list      # [{action, result, conf}, ...]
    risk:            SafetyResult
    tools:           ToolResult
    evidence:        dict      # {rule, semantic, context, tool, safety} 审计全链路
    decision:        str       # "route"|"block"|"confirm"|"options"

# 汇总逻辑(优先级: 安全 > 意图消歧义 > 工具选择):

# Step 1: 安全检查(最先执行, 可短路)
if safety.risk_level == "critical":
    return block("高风险操作, 已拦截: " + safety.block_reason)
if safety.risk_level == "high":
    return confirm(safety.block_reason)

# Step 2: 意图消歧义(融合多路证据)
# 权重: 语义70% + 规则20% + 上下文修正10%
base_intent = semantic (confidence=0.95)
if context.correction:
    # 上下文修正可以翻转意图
    base_intent = merge_with_context(base_intent, context.correction)
if rule_intent != semantic_intent:
    # 规则和语义不一致 → 降低置信度
    confidence *= 0.7

# Step 3: 工具选择(按置信度排序)
tools = match_skills(base_intent, project_status)
if tools.best_confidence < 0.5:
    return options()  # 低置信度 → 出多选项

# Step 4: 输出
return PipelineResult(
    intent=base_intent,
    plan=build_plan(base_intent, tools, safety),
    risk=safety,
    tools=tools,
    evidence={"rule": rule, "semantic": semantic, "context": context, ...},
    decision="route",
)

# 并行调用: 5个模块中只有 semantic 需要 LLM, 其他 4 个都是规则计算(同步)
# 执行顺序:
#   1. 同时启动: rule + context + safety + tools (4个0延迟)
#   2. 同时启动: semantic (LLM, ~2s)
#   3. 等 semantic 完成 → 汇总决策
```

### 4.1 SkillEntry 扩展

```python
# registry/skill_registry.py

@dataclass
class SkillEntry:
    name:         str           # "builder_agent"    (内部标识)
    intent_tags:  list[str]     # ["建站", "生成"...]
    handler:      Callable      # builder_agent_handler
    is_graph:     bool          # 是否 Graph Agent
    description:  str           # "代码生成: 基于需求文档生成/修改HTML"
    display_name: str = ""      # 🆕 "建站小胡"        (前端显示)
    avatar:       str = ""      # 🆕 "⚡"             (前端头像)
    role:         str = ""      # 🆕 "代码生成"        (角色标签)

    def to_dict(self) -> dict:
        """供 /agents 端点序列化"""
        return {
            "id": self.name,
            "name": self.display_name or self.name,
            "avatar": self.avatar,
            "role": self.role,
            "description": self.description,
        }

class SkillRegistry:
    ...
    @classmethod
    def list_agents(cls) -> list[dict]:
        """动态生成 Agent 列表(替代 agents.py)"""
        return [e.to_dict() for e in cls._entries.values()]
```
